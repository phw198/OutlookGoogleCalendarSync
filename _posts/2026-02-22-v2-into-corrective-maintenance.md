With the new major release of OGCS v3, which brings support for Outlook Online and no longer requires the Outlook Classic client, OGCS v2 is entering corrective maintenance mode.  

Rest assured, v3 will still work as it did before with the classic desktop application, if it is installed.

### Corrective Maintenance

Corrective software maintenance involves addressing faults and errors (commonly known as bugs) in application code. This type of maintenance is often initiated upon [receiving bug reports](https://github.com/phw198/OutlookGoogleCalendarSync/wiki/Reporting-Problems) from users who have encountered issues while using the software, which is crucial for ensuring that the software operates smoothly and efficiently. 

This means OGCS v2 will continue to receive bug fixes that aim to eradicate existing bugs, but this is in sharp contrast to development that introduces new features and adapts to a changing IT "landscape" (when Google or Microsoft add news features themselves, for example).

### New Features

Focus will switch to OGCS v3, which will receive new features from now onwards. Bugs that affect both v2 and v3 will be released into both products, but v2 is now essentially static in terms of feature set. 

It is therefore adviseable to upgrade to v3 in the near future, to continue benefiting from the continued development going into the product.
{: .tip}

### Continued Usage of v2

You can absolutely choose to remain on v2 if that contains all the functionality you require. Update notifications will still be shown for v3 releases, but you can choose to skip these and continue accepting v2 releases. 

For example, if you are currently running beta v2.12.0 and have opted in to alpha releases, you will receive a notificaton for the release of alpha v3.0.1. If you choose to skip this release, when alpha v2.12.1 is published, you can then choose to upgrade to this, thus remaining on v2.
